The Big Story: State-Sponsored Poisoning of the Axios Ecosystem On March 31, 2026, the North Korean state-sponsored threat actor Sapphire Sleet successfully compromised the distribution of Axios, one of the most widely used JavaScript HTTP clients. The attackers published two malicious versions to the npm registry—1.14.1 and 0.30.4. While the malicious packages have since been removed from the registry, the reach of Axios within enterprise CI/CD pipelines and production web applications means millions of users were potentially exposed during the infection window. The primary risk lies in the automated nature of modern development environments. Organizations utilizing automated dependency management or "wildcard" versioning likely pulled the poisoned updates immediately upon release. Once integrated, the malicious code establishes communication with Command and Control (C2) infrastructure attributed to Sapphire Sleet. This allows the threat actor to exfiltrate sensitive data, such as environment variables, API keys, and credentials, or to pivot deeper into the developer’s local network and cloud environments. Technically, the compromise functioned as a staged infection. The malicious npm packages contained scripts that, upon installation or execution, triggered a secondary download from an external C2 server. By offloading the primary malicious payload to a second stage, the attackers managed to keep the initial npm package footprint small, potentially evading basic static analysis tools.
