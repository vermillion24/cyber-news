Daily Cyber Intelligence Brief Date: April 4, 2026 To: Technical Leadership and Security Operations Prepared by: Senior Cyber Security Researcher

1. Main Headline#

Critical FortiClient EMS Pre-Auth Bypass (CVE-2026-35616) Under Active Exploitation; Emergency Patches Issued

2. The Big Story: FortiClient EMS Pre-Authentication API Bypass#

Fortinet has issued an emergency out-of-band patch for a critical vulnerability in FortiClient Enterprise Management Server (EMS), tracked as CVE-2026-35616 (CVSS 9.1). The flaw allows unauthenticated remote attackers to bypass API access controls and gain elevated privileges on the management server. This vulnerability is currently being exploited in the wild, targeting organizations that rely on EMS for managing endpoint security posture across large-scale networks.

The risk profile for this vulnerability is severe because FortiClient EMS acts as a central hub for security policy enforcement. By achieving privilege escalation, an attacker can potentially modify security configurations, deploy malicious updates to connected endpoints, or pivot deeper into the corporate infrastructure. The exploitation occurs before any authentication is required, making internet-facing EMS instances high-priority targets for automated scanning and state-sponsored intrusion sets.

Technically, the vulnerability is classified as CWE-284 (Improper Access Control). The flaw resides in the EMS API handler, where specific requests do not undergo sufficient authorization checks. An attacker can craft a sequence of API calls that trick the system into granting administrative access. Fortinet has confirmed that the exploitation is occurring in targeted attacks, and administrators are urged to apply the emergency updates immediately to prevent full system compromise.

3. Vendor Security Watch#

  • Fortinet: Released emergency patches for FortiClient EMS to remediate CVE-2026-35616, an improper access control vulnerability allowing pre-authentication privilege escalation.
  • Microsoft: Issued mitigation guidance for the Axios NPM supply chain compromise (attributed to DPRK-linked Sapphire Sleet). Also released technical analysis on stealthy, cookie-controlled PHP webshells that leverage php-fpm and cron-based persistence in Linux environments.
  • Cloudflare: Launched EmDash beta, a serverless CMS built on Astro 6.0 that attempts to solve plugin-based security risks by running all third-party code in sandboxed Worker isolates.
  • Cisco Talos: Published analysis on the Axios NPM incident and the broader trend of supply chain insecurity. They highlighted that malicious payloads were delivered via actor-controlled infrastructure after a takeover of the Axios package registry.
  • Oracle: No critical updates or security bulletins were included in today’s raw data analysis.

4. Critical Headlines#

  • DPRK Targets JavaScript Ecosystem: The North Korean state actor Sapphire Sleet compromised the widely-used axios NPM package, injecting malicious code into new updates to facilitate C2 communications.
  • React2Shell Exploitation: Threat actors are currently exploiting CVE-2025-55182 in Next.js applications to automate credential theft at scale.
  • Geopolitical Wiper Activity: Two separate wiper campaigns are active; “CanisterWorm” is targeting cloud services in Iran, while an Iran-backed group has claimed a data-wiping attack against medical technology firm Stryker.
  • Hardware Vulnerability Expansion: New research indicates “Rowhammer” bit-flipping attacks have evolved to target GDDR memory in modern NVIDIA GPUs, potentially allowing attackers to bypass system-level memory protections.
  • IoT Botnet Takedown: Federal authorities disrupted the infrastructure for four major botnets (Aisuru, Kimwolf, JackSkid, and Mossad) responsible for record-breaking DDoS attacks involving over 3 million compromised devices.

5. Admin Priority List#

  1. Immediate Patch: Update FortiClient EMS to the latest version to mitigate CVE-2026-35616. Ensure the management interface is not exposed to the public internet.
  2. Dependency Audit: Conduct an immediate audit of all internal projects using the axios NPM package. Revert to known-good versions and monitor for unauthorized outbound C2 traffic.
  3. Next.js Remediation: Verify the version of Next.js in use across the organization; if vulnerable to CVE-2025-55182, apply the React2Shell patch to prevent automated credential harvesting.
