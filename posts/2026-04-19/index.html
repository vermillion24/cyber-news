Daily Cyber Intelligence Brief#

April 2026

Main Headline: Russian GRU Exploits Edge Infrastructure to Hijack 18,000 Corporate Networks via Token Theft#

The Big Story: Stateless Authentication Hijacking via Compromised Edge Routers#

Hackers linked to Russian military intelligence (GRU) have executed a massive campaign targeting legacy internet-facing routers to harvest Microsoft Office authentication tokens. Unlike traditional intrusions that rely on deploying malware to endpoints, this campaign operates entirely at the infrastructure layer. By exploiting known vulnerabilities in older routing hardware, attackers position themselves to intercept unencrypted or poorly secured traffic, specifically targeting session cookies and OAuth tokens used for cloud-based productivity suites.

The primary risk profile includes any organization maintaining end-of-life (EoL) or unpatched edge networking equipment. Because the threat actors are not executing code on the local workstations, standard Endpoint Detection and Response (EDR) solutions are largely blind to the activity. The scale is significant, with security researchers identifying over 18,000 compromised networks where authentication tokens were silently siphoned to facilitate unauthorized access to sensitive corporate data.

Technically, the mechanism utilizes the compromised router as a transparent proxy or traffic analyzer to capture valid session tokens. These tokens are then replayed by the attackers to gain access to Microsoft Office environments, effectively bypassing Multi-Factor Authentication (MFA). Since the tokens represent an already-authenticated session, the “living-off-the-land” approach allows the GRU to maintain persistence and move laterally within cloud environments without triggering traditional identity-based alerts or requiring password cracking.

Vendor Security Watch#

  • Microsoft: Released the April 2026 Patch Tuesday update addressing 167 vulnerabilities. Critical fixes include a SharePoint Server zero-day and a publicly disclosed flaw in Windows Defender dubbed “BlueHammer.”
  • Cisco: Disclosed and patched one vulnerability in Foxit Reader and six vulnerabilities in the LibRaw file reader library to prevent arbitrary code execution via malformed files.
  • Fortinet: FortiGuard Labs reported on the Nexcorium botnet (a Mirai variant) actively exploiting CVE-2024-3721 (CVSS 6.3) to hijack TBK DVR devices for DDoS operations.
  • Cloudflare: Introduced “Redirects for AI Training” to enforce canonical content ingestion by verified crawlers, mitigating the risk of AI agents training on deprecated or sensitive “soft-deleted” data.
  • Oracle: No critical vulnerability disclosures or patches were reported in the current data set.

Critical Headlines#

  • Critical Protobuf.js RCE: A remote code execution flaw in the protobuf.js library has been disclosed. Proof-of-concept (PoC) exploit code is now public, threatening Node.js environments that utilize Google’s Protocol Buffers.
  • Nexcorium Botnet Expansion: Threat actors are leveraging CVE-2024-3721 and EoL TP-Link Wi-Fi router flaws to expand the Nexcorium Mirai-variant botnet for large-scale DDoS attacks.
  • PowMix Botnet Targeting Czechia: A previously undocumented botnet, “PowMix,” has been identified targeting the Czech workforce since December 2025; technical analysis suggests a focus on persistent credential harvesting.
  • REvil Leader Doxed: German authorities have identified “UNKN,” the alleged head of the REvil and GandCrab ransomware gangs, as 31-year-old Russian national Daniil Maksimovich Shchukin.
  • Grinex Exchange Shutdown: The Kyrgyzstan-based cryptocurrency exchange Grinex has suspended operations following a $13.74M hack, which the exchange attributes to Western intelligence agency intervention.

Admin Priority List#

  1. Deploy Microsoft April Patches: Prioritize SharePoint Server and Windows Defender updates to mitigate the zero-day and “BlueHammer” vulnerabilities.
  2. Harden Edge Infrastructure: Conduct an immediate audit of all internet-facing routers; patch or decommission EoL hardware to prevent GRU-linked token harvesting.
  3. Remediate Protobuf.js: Audit software supply chains for protobuf.js dependencies and update to the latest patched version to block published RCE exploits.
