Main Headline: Critical CVSS 10.0 Vulnerability in Flowise AI Faces Mass Exploitation#

The Big Story: Flowise AI Remote Code Execution (CVE-2025-59528)#

A maximum-severity vulnerability, CVE-2025-59528 (CVSS 10.0), is currently under active exploitation targeting Flowise, an open-source platform used for building LLM-based AI agents. The flaw is a code injection vulnerability located within the CustomMCP (Model Context Protocol) node. This node is designed to allow users to input configuration settings for connecting to external data sources; however, a lack of input validation enables threat actors to inject and execute arbitrary code on the underlying server.

Approximately 12,000 Flowise instances are currently exposed to the public internet, many of which are integrated into enterprise AI workflows. Organizations utilizing Flowise to orchestrate internal data movements or automate business processes via LLMs are at extreme risk. Because Flowise often requires high-level permissions to interact with various databases and API endpoints, a successful compromise via RCE provides an attacker with a high-privilege foothold inside the internal network.

The technical mechanism involves the manipulation of configuration parameters within the CustomMCP node. By submitting specially crafted payloads through the UI or API, attackers bypass security checks to execute system commands. Research indicates that exploitation is straightforward and is being leveraged by threat actors to gain initial access, exfiltrate environment variables (including API keys for OpenAI, Anthropic, and AWS), and establish persistence within containerized environments.

Vendor Security Watch#

  • Cloudflare: Launched “Organizations” into public beta. This update consolidates authorization systems to enable centralized management across multiple enterprise accounts, aimed at reducing configuration drift and identity-based misconfigurations.
  • Cisco: Talos reported on a supply chain compromise involving the Axios NPM package. The incident involved malicious payloads delivered from actor-controlled infrastructure. Additionally, Talos warned of the “democratization” of Business Email Compromise (BEC) through automated toolsets.
  • Microsoft: Issued a warning regarding Storm-1175, a high-velocity ransomware actor deploying Medusa ransomware. The actor weaponizes recently disclosed vulnerabilities in web-facing assets for initial access. Microsoft also detailed a new AI-enabled device code phishing campaign that automates the generation of live authentication codes to bypass MFA.
  • Fortinet: FortiGuard Labs identified a DPRK-linked campaign targeting South Korea. The attack uses obfuscated LNK files to drop decoy PDFs while utilizing GitHub as a Command-and-Control (C2) infrastructure to evade traditional network-level detection.
  • Oracle: No critical security patches or vulnerability disclosures were reported in today’s data set.

Critical Headlines#

  • Windows Zero-Day Leak: A disgruntled researcher has leaked exploit code for “BlueHammer,” an unpatched privilege escalation flaw in Windows that allows attackers to transition from standard user to SYSTEM permissions.
  • GPUBreach Attack: Researchers have demonstrated a new Rowhammer-style attack targeting GDDR6 memory. Dubbed “GPUBreach,” the exploit induces bit-flips in GPU memory to escalate privileges and achieve full system takeover.
  • FBI “Major Incident”: The FBI confirmed a significant breach of its surveillance management networks, used for wiretap operations. The intrusion is classified as a “major incident,” suggesting deep compromise of sensitive law enforcement data.
  • REvil/GandCrab Leader Doxed: German authorities have identified 31-year-old Russian national Daniil Maksimovich Shchukin (aka UNKN) as the operator behind the REvil and GandCrab ransomware syndicates.
  • CanisterWorm Wiper: A new worm is targeting Iranian infrastructure, specifically wiping systems that use Farsi language settings or Iran-specific time zones, spreading via misconfigured cloud services.

Admin Priority List#

  1. Immediate Patch: Update all Flowise AI instances to the latest version to mitigate CVE-2025-59528. Ensure these instances are not directly exposed to the internet without a VPN or Zero Trust gateway.
  2. Monitor Windows Logs: Deploy detections for unauthorized privilege escalation attempts targeting the “BlueHammer” exploit. Monitor for unusual processes spawning with SYSTEM integrity.
  3. Harden M365 Auth: Disable Device Code Flow if not required, or restrict it to managed devices to counter the new AI-automated phishing campaigns reported by Microsoft.
