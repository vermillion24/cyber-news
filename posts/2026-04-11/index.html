Daily Cyber Intelligence Brief#

Main Headline: Russian Military Intelligence Compromises 18,000 Networks via Legacy Router Exploitation

The Big Story Hackers linked to Russian military intelligence (GRU) have weaponized known vulnerabilities in legacy SOHO and enterprise routers to execute a massive, global authentication token harvesting campaign. Unlike traditional attacks that rely on deploying malware to endpoints, this operation targets the network layer to intercept Microsoft Office session tokens directly from transit. By compromising the routing hardware, threat actors can bypass traditional Endpoint Detection and Response (EDR) systems that are blind to traffic-level manipulation at the edge.

The campaign has successfully siphoned credentials from over 18,000 networks. The primary risk is the bypass of Multi-Factor Authentication (MFA); because the stolen tokens represent already-authenticated sessions, the attackers can replay them to gain immediate, unauthorized access to cloud environments, including Outlook, SharePoint, and OneDrive. Organizations using unpatched or end-of-life (EoL) networking hardware are at the highest risk, as these devices often lack modern security telemetry and are rarely audited for unauthorized configuration changes.

Technically, the threat actors establish persistence on the compromised routers to perform man-in-the-middle (MitM) or traffic redirection. By siphoning session secrets as they traverse the device toward Microsoft authentication endpoints, the attackers maintain a “malwareless” footprint. This strategy allows for long-term data exfiltration and lateral movement within cloud suites while remaining invisible to security teams that focus exclusively on host-based logging and ignore anomalous traffic patterns originating from the network infrastructure.

Vendor Security Watch

  • Microsoft: Released a technical breakdown of a severe Intent Redirection vulnerability in a popular third-party Android SDK that exposes sensitive data in millions of financial and wallet apps. Separately, Microsoft addressed a researcher-led “Recall” bypass tool, though it officially maintains the tool does not exploit a new vulnerability.
  • Cloudflare: Announced a revised roadmap targeting full post-quantum security by 2029. Technically, they introduced a method using symbolic execution and the Z3 theorem prover to automate the generation of BPF-based “magic packets” for malware analysis, reducing analysis time from hours to seconds.
  • Cisco: Talos researchers issued a warning regarding the “collapse of the patch window,” highlighting that the speed of attacker exploitation, fueled by AI and automation, is now outpacing human-scale remediation cycles.
  • Fortinet / Oracle: No significant technical vulnerability disclosures or critical security patches were reported in today’s data cycle.

Critical Headlines

  • Critical Infrastructure Risk: Nearly 4,000 Rockwell Automation PLCs in the US are currently exposed to the internet, creating a target-rich environment for Iranian-linked cyberattackers.
  • GlassWorm Campaign: A new Zig-based dropper is being distributed via the “specstudio.code-wakatime-activity-tracker” Open VSX extension, designed to infect multiple developer IDEs.
  • Compiler Security: Go versions 1.26.2 and 1.25.9 have been released to fix two critical memory safety bugs in the compiler that could result in memory-unsafe programs.
  • Wiper Attack: The “CanisterWorm” malware is actively targeting Iranian cloud services, specifically wiping data on systems where the local time zone is set to Iran or the language is Farsi.
  • GitLab Patches: Multiple vulnerabilities were addressed in GitLab, including flaws that could lead to remote Denial of Service (DoS) and unauthorized data manipulation.

Admin Priority List

  1. Audit Edge Infrastructure: Immediately identify and patch all internet-facing routers; replace any hardware that has reached End-of-Life to prevent session token siphoning.
  2. Sanitize Developer IDEs: Conduct a sweep for the “specstudio.code-wakatime-activity-tracker” extension and implement a whitelist policy for VS Code/IDE marketplaces.
  3. Update Go Environments: Recompile and redeploy critical internal Go-based applications using Go 1.26.2 or 1.25.9 to mitigate compiler-level memory safety risks.
